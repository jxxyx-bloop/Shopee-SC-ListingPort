The Challenge

A Strategic Seller Segment, Blocked by Operational Friction

Japanese Cross-Border merchants create accounts in each market's local Seller Center (MY, SG, TH, PH) — no dedicated JP Seller Center exists today.

Their product listings are already in English and ready to cross-post. But today, sellers manually re-enter every product field into each target market — names, descriptions, prices, images, variations, shipping info.

This is error-prone, slow (hours per batch), and creates a scaling bottleneck for a high-value, growth-minded seller segment.

The ideal fix — a Global Seller Center — has low ROI for this niche pool. We need a practical bridge that works today.

Hours → Minutes Time to duplicate a product batch across markets
~500 listings Processed per single transfer session
5 files → 1 file Complexity absorbed entirely by the tool

Decision Framework

Why This Approach, Not the Others

We evaluated four approaches across feasibility, maintenance cost, time-to-deploy, and evolutionary potential. Here is why we chose what we chose.

Option A: Status Quo

Manual copy-paste, field by field

Pros

  • No development effort
  • No technical risk

Cons

  • Hours per batch, high error rate
  • Scales with headcount, not technology
  • Seller frustration and churn risk
Not scalable — this is the problem

Option B: Browser Automation / RPA

Selenium, UiPath — automate SC clicks

Pros

  • No API dependency
  • Could automate end-to-end in theory

Cons

  • Any Seller Center UI change breaks it
  • Login/session handling is fragile
  • Constant maintenance burden
  • Not audit-friendly, ToS risk
High cost, fragile, risky

Option C: Shopee Open API

Pull listings via API, push to target market

Pros

  • Most elegant, fully automated
  • Real-time, no manual file handling
  • Production-grade at scale

Cons

  • Internal API access not available to our team today
  • Requires OAuth per seller, more infrastructure
  • Significantly longer time to deploy
Ideal long-term — blocked today

Option D: File-Based Transformation

Leverage existing Mass Upload/Download workflow

Pros

  • Works within existing SC capabilities
  • Zero new infrastructure or permissions
  • Fast to deploy (days, not quarters)
  • Seller-familiar workflow
  • Extensible to new markets via config

Cons

  • Manual trigger per batch (not real-time)
  • Category IDs mapped manually per market pair
Right tool for right now — chosen

Delivered Solution

What We Built

Phase 1 — CLI Tool

Python Command-Line Engine

Audience: Technical ops staff, power users

  • Reads 5 Mass Update export files from source market
  • Merges product data by Product ID (handles multi-variation products)
  • Converts prices via configurable exchange rates
  • Maps category IDs via JSON configuration
  • Outputs a valid, upload-ready xlsx for the target market
  • Handles Shopee's non-standard Excel format (a known compatibility issue solved at the engine level)

✓ Production-ready · 60+ automated tests

Phase 2 — Web App

Streamlit Visual Interface

Audience: Non-technical ops staff, seller managers

  • Drag-and-drop file upload with auto-detection of export type
  • Visual product preview table before transformation
  • Interactive category mapping UI (no JSON editing needed)
  • Exchange rate override field
  • One-click transform and instant file download

✓ Production-ready · Deployable on any laptop or internal server

Evolution Path

Future State — With Shopee Open API

Current State (File-Based)

  • Seller downloads 5 export files from source SC
  • Uploads files to web app
  • Reviews product preview and maps categories
  • Downloads transformed output file
  • Uploads output file to target market SC

Works today. No new permissions needed.

Future State (API-Connected)

  • Tool pulls listings directly via API — no file download
  • AI-powered category matching across markets
  • Auto-submit to target market (with seller approval step)

  • Multi-market dashboard: one view, all markets
